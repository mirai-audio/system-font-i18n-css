About

A modern native system font stack for consistent multi-lingual typesetting, optimized by unicode ranges.

system-font-i18n provides twelve variations for the Sans-serif family of system font. This font stack provides more consistent character typesetting across multiple languages and all modern operating systems.

system-font-i18n is optimized to select the best system font on a per-character basis, based on the unicode range of that character.

system-font-i18n provides one font family, sans-serif. The font family has three weights, light (300), regular (400), and bold (700). Each weight has two styles– regular, and italic

Modern operating systems such as macOS (Mavericks+), iOS, Android (2.4+), Windows 7+, and Ubuntu ship with a quality set of system fonts. Currently, css doesn't make it easy to select a high-quality system typeface that displays a consistent font-family in all languages.

This problem is illustrated in this paragraph. The text in this paragraph is set with font-family: serif;. macOS Sierra on Firefox will select "Times Roman" for the English (latin-1 unicode) characters. Within the same paragraph, Japanese characters (あア安) will by typeset in "Hiragino Kaku Gothic ProN W3". In this instance, the English characters are typeset in a serif font, while the Japanese characters are set in a sans-serif/gothic font.

Other projects have solutions to select local system typefaces, such as system-font-css, but they do not solve the problem of inconsistent font-family typesetting in internationalized or multi-language documents.

Usage

General Usage

The robust and preferred method is to reference the system-font-i18n.css stylesheet in the <head> of the <html> document. Then add the utility helper class to the <body> class. 

<link rel="stylesheet" href="system-font-i18n.css">
…
<body class="system-font-i18n-sans">

Another method is to add the system-font-i18n-sans font stack properties directly to the html, body rules in an existing stylesheet: 

html,
body {
  font-family: system-sans-serif,
    system-gothic,
    sans-serif,
    "Twitter Color Emoji", "Apple Color Emoji", "Segoe UI Emoji";
}

system-font-i18n.css is designed to play well with other css frameworks, notably normalize.css, and sanitize.css. system-font-i18n.css introduces only one global CSS rule, .system-font-i18n-sans and two @font-face names: system-sans-serif, system-gothic.

The W3C is currently discussing standarizing a generic "system" font family. As such, system-font-i18n.css has avoided naming our @font-face "system" to prevent future collisions with the standard.

Support

Modern macOS systems since El Capitan will select "San Francisco". Yosemite systems will select "Helvetica Neue". Mavericks back to Kodiak will select "Lucida Grande".

Windows and Windows Phone will select "Segoe UI". XP selects "Arial", not Tahoma (does not have an italic variation).

Android will select "Roboto".

Ubuntu will use the apty named Ubuntu, and other linux variants will select "Oxygen-Sans" or "Cantarell"

This project has optimized unicode typesetting for English (unicode ranges basic-latin, latin-1 supplement, general punctuation, superscripts & subscripts, number forms, arrows, and mathmatical operators), and Japanese (CJK symbols and punctuation, Hiragana, Katakana, CJK unified ideographs). The project is accepting help in supporting other languages.
