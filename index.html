A beautiful modern native system font stack for rendering multilingual text, consistent across all major desktop and mobile operating systems and browsers

The font stack provides 5 weights (Thin, Light, Regular, Bold, Heavy) and supports Italic variants for each weight. This font stack provides more consistent per-character typesetting across western and asian languages on all major modern desktop and mobile operating systems and browsers.

Modern operating systems such as macOS (Mavericks+), iOS, Android (2.4+), Windows 7+, and Ubuntu ship with a quality set of system fonts. Currently, css doesn't make it easy to select a high-quality system typeface that displays a consistent font-family in all languages.

This problem is illustrated in this paragraph. The text in this paragraph is set with font-family: serif;. macOS Sierra on Firefox will select "Times Roman" for the English (latin-1 unicode) characters. Within the same paragraph, Japanese characters (あア安) will by typeset in "Hiragino Kaku Gothic ProN W3". In this instance, the English characters are typeset in a serif font, while the Japanese characters are set in a sans-serif/gothic font.

Other projects have solutions to select local system typefaces, such as system-font-css, but they do not solve the problem of inconsistent font-family typesetting in internationalized or multi-language documents.

Usage

General Usage

The robust and preferred method is to reference the system-sans-serif.css stylesheet in the <head> of the <html> document. Then add the utility helper class to the <body> class. 

<link rel="stylesheet" href="system-sans-serif.css">
…
<body class="system-sans-serif">

system-sans-serif.css is designed to play well with other css frameworks, notably normalize.css, and sanitize.css.

The W3C is currently discussing standarizing a generic "system-ui" font family. As such, system-font-i18n.css has avoid using a @font-face named "system-ui" to prevent future collisions with the standard.

Support

Modern macOS systems since El Capitan will select "San Francisco". Yosemite systems will select "Helvetica Neue". Mavericks back to Kodiak will select "Lucida Grande".

Android and Chrome OS will select "Roboto" and render Chinese, Japanese, and Korean characters using "Noto Sans CJK JP".

Ubuntu will use "Ubuntu", and other linux variants will select "Oxygen-Sans" or "Cantarell" for KDE and Gnome.

Windows and Windows Phone will select "Segoe UI". Windows XP selects "Arial", (not Tahoma, which does not have an italic variation).

This project has optimized unicode typesetting for English (unicode ranges basic-latin, latin-1 supplement, general punctuation, superscripts & subscripts, number forms, arrows, and mathmatical operators), and Japanese (CJK symbols and punctuation, Hiragana, Katakana, CJK unified ideographs).

  9. 0.1.0: Initial release
